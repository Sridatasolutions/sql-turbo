**1.** OLTP systems are optimized for:
- A) Analytics
- B) Transactions (many small reads/writes)
- C) Reporting
- D) Archiving

**2.** OLAP systems are optimized for:
- A) Order entry
- B) Analytical queries over large datasets

- D) Inventory updates

**3.** In a star schema, the central table is the:
- A) Dimension table
- B) Fact table
- C) Bridge table
- D) Lookup table

**4.** Dimension tables store:
- A) Measures like revenue
- B) Descriptive attributes (who/what/where/when)
- C) Only keys
- D) Indexes

**5.** A surrogate key is:
- A) The natural business key
- B) A system-generated key independent of business data
- C) A foreign key only
- D) A composite of all columns

**6.** A fact table typically contains:
- A) Long text descriptions
- B) Foreign keys to dimensions + numeric measures
- C) Only dates
- D) No keys

**7.** Denormalization in warehouses is done to:
- A) Save storage
- B) Speed up analytical reads (fewer joins)
- C) Enforce integrity
- D) Reduce duplication

**8.** A date dimension lets you analyze by:
- A) Only exact timestamps
- B) Year, quarter, month, weekday, holiday flags
- C) Primary keys
- D) Random groupings
