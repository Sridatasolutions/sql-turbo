# 🧩 Mission 8 Quiz — Set Operations (UNION / INTERSECT / EXCEPT)

Answers at the bottom.

---

**1.** `UNION` does what?
- A) Joins tables side by side
- B) Stacks rows from two queries and removes duplicates
- C) Multiplies rows
- D) Filters rows

**2.** Difference between `UNION` and `UNION ALL`?
- A) None
- B) `UNION ALL` keeps duplicates (and is faster)
- C) `UNION` keeps duplicates
- D) `UNION ALL` removes more rows

**3.** `INTERSECT` returns:
- A) Rows in either query
- B) Rows present in BOTH queries
- C) Rows in the first only
- D) All rows

**4.** `EXCEPT` returns:
- A) Rows in both
- B) Rows in the first query that are NOT in the second
- C) Rows in the second only
- D) Duplicates

**5.** For set operations to work, the queries must have:
- A) The same table
- B) The same number of columns with compatible types
- C) A JOIN
- D) The same WHERE

**6.** Which is generally faster when you know there are no duplicates?
- A) `UNION`
- B) `UNION ALL`
- C) `INTERSECT`
- D) `EXCEPT`

**7.** To find department_ids appearing in BOTH `employees` and `finance_budget`, use:
- A) `UNION`
- B) `INTERSECT`
- C) `EXCEPT`
- D) `JOIN ALL`

**8.** Set operations differ from JOINs because they:
- A) Combine columns horizontally
- B) Combine rows vertically (stack result sets)
- C) Cannot use SELECT
- D) Require indexes
